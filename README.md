# MHXY PK Vision Assistant

梦幻西游手游 PK 合规视觉辅助/训练助手研究项目。

## 重要边界

本项目不开发、也不接受以下能力：

- 读取、篡改或注入游戏客户端内存/网络封包/进程；
- 绕过反外挂、自动化代打、自动决策或自动操作；
- 显示游戏原本不向玩家公开的对方真实血量、蓝量、技能冷却、状态等隐藏信息；
- 任何违反《梦幻西游》手游、网易游戏用户协议或赛事规则的插件。

用户最初想做“PK 时看到对方血条”的插件。该目标如果意味着获取游戏未公开的真实数值，属于高风险作弊/外挂方向，本仓库将其重构为合规方向：

1. 赛后复盘：导入录像/截图，人工标注或 OCR 已显示信息，估算战斗节奏与伤害区间。
2. 训练沙盒：用手动输入或公开可见事件构建血量区间估计器，帮助理解集火、守尸、回复窗口。
3. 观战/直播辅助：只在获得授权的录像、直播画面或自有素材上做非侵入式屏幕分析。
4. 无游戏交互：工具不向游戏客户端发送输入，不读取进程，不连接游戏网络。

## MVP 方向

第一版建议做“PK 录像复盘 + 血量区间估算器”：

- 输入：本地视频/截图、人工录入基础属性、回合事件表。
- 输出：每个单位的血量区间、伤害/治疗事件时间线、关键回合复盘摘要。
- 技术：Python、OpenCV/Pillow、可选 OCR、Streamlit/FastAPI 前端。

## MuMu 模拟器 / macOS 场景

当前目标运行环境是 macOS 上的 MuMu 模拟器。项目可以复用已有 `JKevinXu/GameAutomation` 师门任务库中“只读视觉识别”的经验，但不能复用自动点击/自动任务执行能力。

可复用经验：

- MuMu 应用路径通常是 `/Applications/MuMuPlayer.app`。
- macOS 需要给 Terminal/运行器授予 Screen Recording 权限，才能截图或录屏。
- Retina 屏幕要区分逻辑坐标和物理截图坐标，缩放因子可用“截图尺寸 / pyautogui 屏幕尺寸”估算。
- OpenCV 模板匹配适合识别公开可见 UI 元素，例如回合数、按钮、状态图标、单位头像区域。
- 模板素材可按 `assets/templates/mumu/` 管理，并记录分辨率、缩放因子、模拟器窗口尺寸。

合规限制：

- MVP 默认用 MuMu 窗口截图、macOS 录屏、OBS/QuickTime 录屏作为输入。
- 第一阶段只做离线导入、抽帧、ROI 标注、血量区间估算。
- 不读取 MuMu/游戏进程内存，不 hook 渲染层，不抓包，不用 ADB，不自动点击或自动战斗。
- 如果后续做实时叠加，也只能是只读公开画面标注，并且明确展示“估算区间，不是真实血量”。
