# MHXY PK Vision Assistant MVP Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Build a compliant offline PK replay analysis MVP that estimates HP intervals from manually entered visible battle events.

**Architecture:** Local-first Python package with strict compliance guards, typed battle models, an interval engine, and optional Streamlit UI. The MVP never reads the game client, memory, packets, or hidden state.

**Tech Stack:** Python 3.10+, Pydantic, pytest, optional Streamlit/OpenCV/Pillow.

---

### Task 1: Define typed battle models

**Objective:** Create unit and event models for replay analysis.

**Files:**
- Create: `src/mhxy_pk_vision_assistant/models.py`
- Test: `tests/test_models.py`

**Steps:**
1. Add Pydantic models: `HpRange`, `Unit`, `BattleEvent`, `EventType`.
2. Validate `low <= high`, non-negative values, and event turn >= 1.
3. Write tests for valid and invalid ranges.
4. Run: `pytest tests/test_models.py -v`.
5. Commit: `feat: add battle data models`.

### Task 2: Implement HP interval engine

**Objective:** Calculate current HP ranges from visible events.

**Files:**
- Create: `src/mhxy_pk_vision_assistant/interval_engine.py`
- Test: `tests/test_interval_engine.py`

**Steps:**
1. Write failing tests for damage, healing, down state, revive state.
2. Implement `apply_event(state, event)` and `run_timeline(units, events)`.
3. Ensure outputs include rationale strings.
4. Run: `pytest tests/test_interval_engine.py -v`.
5. Commit: `feat: add hp interval engine`.

### Task 3: Add JSON import/export

**Objective:** Load and save local match analysis files.

**Files:**
- Create: `src/mhxy_pk_vision_assistant/io.py`
- Test: `tests/test_io.py`
- Create: `examples/simple_match.json`

**Steps:**
1. Define `load_match(path)` and `save_match(match, path)`.
2. Add sample JSON using the docs data model.
3. Test round-trip serialization.
4. Run: `pytest tests/test_io.py -v`.
5. Commit: `feat: add match json io`.

### Task 4: Generate Markdown report

**Objective:** Export a readable replay report.

**Files:**
- Create: `src/mhxy_pk_vision_assistant/report.py`
- Test: `tests/test_report.py`

**Steps:**
1. Write tests that report contains match id, unit ranges, event timeline, and disclaimers.
2. Implement `render_markdown_report(match, timeline)`.
3. Include the wording “估算区间，不代表真实隐藏血量”.
4. Run: `pytest tests/test_report.py -v`.
5. Commit: `feat: add markdown report export`.

### Task 5: Optional local UI

**Objective:** Provide a Streamlit UI for manual replay annotation.

**Files:**
- Create: `app/streamlit_app.py`
- Modify: `pyproject.toml`

**Steps:**
1. Add upload/load JSON controls.
2. Add forms for unit and event entry.
3. Show timeline and report preview.
4. Keep UI offline-only; no game process access.
5. Run: `streamlit run app/streamlit_app.py` for manual verification.
6. Commit: `feat: add local streamlit annotation ui`.

### Task 6: Add compliance CI

**Objective:** Prevent forbidden feature descriptions from entering docs/examples accidentally.

**Files:**
- Create: `scripts/check_compliance_terms.py`
- Create: `.github/workflows/ci.yml`

**Steps:**
1. Scan project docs for forbidden terms unless they appear in explicit “non-goal/forbidden” sections.
2. Run pytest and ruff in CI.
3. Run locally: `pytest && ruff check .`.
4. Commit: `ci: add tests and compliance checks`.
