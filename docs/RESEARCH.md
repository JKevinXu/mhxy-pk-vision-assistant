# 调研报告：梦幻西游手游 PK 血条/视觉辅助的边界与可行方向

## 1. 原始需求

“开发一个梦幻西游 PK 时候看到对方血条的插件”。

该需求需要先拆分：

- A. 获取并显示游戏原本不展示的对方真实血量：高风险，通常意味着读取客户端内存、解析网络封包、逆向协议或绕过反外挂。
- B. 基于玩家可见画面、录像、战斗日志/人工录入做赛后复盘与血量区间估算：可作为合规训练工具。
- C. 直播/观战场景对已授权画面做 OCR/标注：可作为可访问性或内容生产工具，但不能揭示隐藏状态。

本项目只采用 B/C，不采用 A。

## 2. 合规与平台风险

公开检索显示，网易游戏与《梦幻西游》手游相关协议、反外挂专题均将外挂、作弊、第三方辅助软件、破坏公平性的行为列为禁止或处理对象。相关入口包括：

- 《梦幻西游》手游用户协议：`https://my.163.com/2014/12/18/17197_484611.html`
- 网易游戏最终用户使用许可协议：`https://game.163.com/app/client/userAgreement.html`
- 网易游戏反外挂专题：`https://game.163.com/topic/315fwg/`

因此，任何“真实对方血量显示”如果依赖隐藏数据、客户端注入或封包解析，都不应实现。

## 3. 同类技术参考

公开资料中有多类与“游戏画面辅助”相关但边界不同的项目：

- 实时游戏翻译/OCR：如 RSTGameTranslation、OCR-Translator、LunaTranslator。它们读取屏幕文字并显示翻译，典型用途是可访问性/语言辅助。
- OBS/直播叠加：如 OBS Input Overlay、ScoreSight 等，主要服务直播、赛事记分牌、可访问性展示。
- 赛后分析工具：常见于电竞复盘，输入录像、事件标注、统计数据，输出复盘报告。

可借鉴的合规技术点：屏幕/视频帧读取、OCR、人工标注、时间线 UI、区间估计、导出报告。

不可借鉴/不可实现的点：读取游戏内存、hook 图形 API、解析网络封包、自动操作、隐藏状态还原。

## 4. 技术可行性评估

### 4.1 录像/截图复盘

可行。输入本地视频/截图；对画面区域做裁剪、OCR 或手工标注；结合事件模型计算血量区间。

优点：低风险、可测试、可开源。
缺点：实时性弱，依赖标注质量。

### 4.2 实时屏幕 OCR 辅助

部分可行，但需要严格限制：只能读取公开可见内容，不能自动操作游戏，不能绕过遮挡或推断隐藏真实值。建议 MVP 暂不做实时模式，先做离线复盘。

### 4.3 真实对方血量插件

不建议也不支持。要实现通常需要隐藏信息来源，存在明显公平性、账号安全、法律与平台封禁风险。

## 5. 推荐产品定位

名称：MHXY PK Vision Assistant

定位：梦幻西游手游 PK 录像复盘与训练助手。

核心能力：

1. 导入录像/截图。
2. 手动录入双方单位、门派、已知血量/抗性/速度等公开或自有信息。
3. 标注每回合伤害、治疗、倒地、召唤、药品、特技等事件。
4. 输出血量区间估计，而非真实隐藏血量。
5. 生成复盘摘要：关键集火窗口、回复压力、守尸/解封/拉人时机。

## 6. MVP 建议

先不做移动端插件，做 Web/本地桌面复盘工具：

- 事件表编辑器；
- 单位与血量区间模型；
- 回合时间线；
- 简单报告导出；
- 合规守卫：任何功能 PR 都必须说明数据来源与是否涉及隐藏状态。
